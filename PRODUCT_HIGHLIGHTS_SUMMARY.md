# 🎓 EduGenius 产品核心亮点总结

> **智能教材学习平台** - AI驱动的个性化教育解决方案

---

## 📋 产品概述

EduGenius 是一款基于人工智能的教材学习平台，通过多Agent系统和大语言模型技术，为学生提供智能化的教材理解、章节划分、个性化教学和自适应测评服务。

**核心价值**：让每本教材都能"说话"，让每个学生都能获得专属的一对一AI导师。

---

## 🌟 核心技术亮点

### 1. 混合文档处理架构（行业首创）

**问题**：大多数教材PDF是扫描版，传统系统无法处理
**解决方案**：智能双路径处理系统

```
┌─────────────┐
│  PDF 上传   │
└──────┬──────┘
       │
       ▼
┌─────────────────────┐
│  智能检测引擎       │
│  • 分析文本层占比   │
│  • 判断文档类型     │
└──────┬──────────────┘
       │
       ├─ 文本占比 ≥ 10% ──→ ⚡ 快速路径（5-10秒）
       │                       • 直接文本提取
       │                       • 100%准确率
       │                       • 实时章节划分
       │
       └─ 文本占比 < 10%  ──→ 🔬 OCR路径（2-10分钟）
                               • PaddleOCR识别
                               • 85-95%准确率
                               • 智能章节提取
                               • 置信度质量检测
```

**技术优势**：
- ✅ 自动判断PDF类型，无需用户手动选择
- ✅ 文本版PDF秒级处理，用户体验极佳
- ✅ 扫描版PDF通过OCR识别，扩大适用范围
- ✅ 置信度质量检测，低质量文档主动警告

**关键指标**：
| 文档类型 | 处理速度 | 准确率 | 适用场景 |
|---------|---------|--------|---------|
| 文本版PDF | 5-10秒 | 100% | 电子书、数字教材 |
| 扫描版PDF | 2-10分钟 | 85-95% | 传统纸质教材扫描件 |

### 2. 多Agent智能教学系统

**架构设计**：基于LangGraph的多Agent协作系统

```
┌─────────────────────────────────────────────────┐
│             EduGenius Multi-Agent System        │
└─────────────────────────────────────────────────┘
                        │
        ┌───────────────┼───────────────┐
        │               │               │
        ▼               ▼               ▼
   ┌─────────┐    ┌─────────┐    ┌─────────┐
   │Architect│    │  Tutor  │    │Examiner │
   │课程设计师│    │  AI导师 │    │  出题官 │
   └────┬────┘    └────┬────┘    └────┬────┘
        │              │               │
        │              │               │
        ▼              ▼               ▼
   学习目标规划    个性化讲解     自适应测评
   知识图谱构建    等级适配      难度动态调整
   章节智能划分    实时问答      薄弱点分析
```

**Agent详解**：

#### 📐 Architect Agent（课程设计师）
- **职责**：分析教材内容，生成结构化学习路径
- **输出**：
  - 章节划分（10+章节精准提取）
  - 学习目标（每章3-5个核心目标）
  - 知识图谱（概念关联关系）
- **技术**：
  - LLM驱动的TOC解析
  - 书签提取算法（PyMuPDF）
  - 智能标题识别

#### 🧠 Tutor Agent（AI导师）
- **职责**：提供个性化讲解和答疑
- **特点**：
  - 5级认知水平适配（L1-L5）
  - 风格差异化（温柔→严厉）
  - 实时流式对话（SSE）
- **技术**：
  - DashScope（通义千问）LLM
  - RAG检索增强（ChromaDB）
  - 等级自适应提示工程

#### 📝 Examiner Agent（出题官）
- **职责**：生成章节测评题目
- **输出**：
  - 选择题、填空题、问答题
  - 难度分级（1-5级）
  - 详细解析
- **技术**：
  - 多维度题目生成（理解力/逻辑力/术语力等）
  - 知识点覆盖度检测
  - 干扰项智能生成

### 3. 5级认知水平自适应系统

**认知模型**：基于布鲁姆教育目标分类法

| 等级 | 名称 | 特征 | 教学风格 | 适合人群 |
|-----|------|------|---------|---------|
| L1 | 基础级 | 知识记忆 | 温柔鼓励型 | 初学者 |
| L2 | 理解级 | 概念理解 | 耐心引导型 | 入门者 |
| L3 | 应用级 | 知识应用 | 标准教学型 | 中级者 |
| L4 | 分析级 | 逻辑分析 | 严格启发型 | 进阶者 |
| L5 | 专家级 | 综合评价 | 严厉挑战型 | 高手 |

**自适应机制**：
```
┌─────────────────────────────────────┐
│       学生学习表现监测              │
│   • 连续答对3题 + 正确率≥85%       │──→ 升级 (L1→L2→L3)
│   • 连续答错2题 + 正确率≤50%       │──→ 降级 (L3→L2→L1)
└─────────────────────────────────────┘
```

**技术亮点**：
- 实时动态调整（无需人工干预）
- 个性化学习路径（因材施教）
- 激励机制（升级成就）

## 📊 核心数据结构

### 数据库架构
```sql
-- 用户表
users (id, email, username, password, cognitive_level)

-- 文档表（含OCR字段）
documents (
    id, md5_hash, filename, title, total_pages, total_chapters,
    processing_status, has_text_layer, ocr_confidence,
    uploaded_by, uploaded_at
)

-- 进度表（含章节状态）
progress (
    id, user_id, document_id, chapter_number, chapter_title,
    status, completion_percentage, cognitive_level_assigned,
    quiz_success_rate, started_at, completed_at
)

-- 对话历史表
conversations (id, user_id, document_id, role, content, created_at)

-- 题目表
questions (id, document_id, chapter_number, question_type,
    question_text, options, correct_answer, explanation, difficulty)

-- 测验尝试记录表
quiz_attempts (id, user_id, progress_id, question_id,
    question_text, user_answer, correct_answer, is_correct)
```

### 向量存储（ChromaDB）
- Collection命名：`{md5_hash}`
- Metadata：`{page, chunk_index, document_id}`
- Embedding模型：DashScope text-embedding-v2

## 🛠️ 技术栈总结

- **日志**：Python logging + JSON格式
